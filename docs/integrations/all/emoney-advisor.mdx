## Access requirements
| Pre-Requisites | Status | Comment|
| - | - | - |
| Paid dev account | ✅ | Requires eMoney API onboarding |
| Paid test account | ✅ | eMoney recommends separate test and production certs |
| Partnership | ✅ | Access is approved by eMoney team |
| App review | ❓ | Confirm during onboarding |

## API gotchas

- eMoney uses certificate-based JWT auth; access is not fully self-service.
- Rate limit is 60 concurrent calls (HTTP 429 on limit exceeded).
- Most planning endpoints are polled (no webhook change feed for ingestion use cases).
